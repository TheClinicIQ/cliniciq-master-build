## The Strategy Layer Handoff — What Gets Delivered

The handoff payload is the complete package every Builder agent receives at activation. It is assembled by the Core Orchestrator Agent from the outputs of the Brand Strategy Agent, Growth Strategy Agent, and Onboarding Agent.

### The Complete Handoff Package

**Document 1 — Expert Dossier Brief (from Layer 1)**
The human-readable narrative of who the expert is, their voice, their avatar, their offer, their philosophy, their proof bank, and their vision. Every Builder agent loads this as foundational context.

*Key sections Builder agents reference most:*
- Voice DNA (sample extracts + all schema fields) — calibration standard for all copy
- Avatar Psychology brief — the narrative description of the primary avatar
- Offer Stack — names, prices, descriptions, value stack, guarantee
- Proof Bank — top 3 transformation stories with full narrative
- Root Cause Map — new vehicle name, mechanism explanation, analogy

---

**Document 2 — Brand North Star Document — Part 2 (from Brand Strategy Agent)**
The operational rules layer. Builder agents load this as their primary guardrail.

*What Builder agents use it for:*
- ALWAYS/NEVER rules — the first check before any output is finalized
- One Big Domino — the belief every piece of content exists to establish or reinforce
- Avatar Quick Reference — demographics, awareness level, core fear, transformation identity, language to use, language to avoid
- Mechanism — name, plain-language explanation, demo concept
- Offer Reference — all four tiers at a glance
- Proof Anchors — the approved case studies (do not fabricate)
- Content Pillars — the strategic purpose of every pillar
- Voice Standard Examples — the calibration targets to match

---

**Document 3 — Monthly Messaging Brief (from Growth Strategy Agent)**
The campaign-level creative brief. Updated monthly. Builder agents read this alongside the Brand North Star.

*What Builder agents use it for:*
- Hook Bank — pull hooks from here, never invent from scratch
- Angle Map — the specific entry points for each pillar this month
- Objection Response Library — when writing close sequences, email follow-ups, sales scripts
- Emotional Beat Sequence — map every asset to a specific point in the journey
- Social Proof Deployment Map — which case study goes where
- Campaign-Specific Language — new framings being tested this month

---

**Document 4 — Monthly Master Plan Brief (from Growth Strategy Agent)**
The coordination brief. Tells every Builder agent what is being built this month, in what order, for which channels.

*What Builder agents use it for:*
- Primary CTA for the month — every asset points here
- Content calendar — which assets are needed and when
- Webinar language variant — which word is being tested (training/masterclass/workshop/etc.)
- Active email sequences — what is running so Builder doesn't duplicate
- New assets needed — the specific build queue for the month

---

**Document 5 — Asset Brief (from Core Orchestrator)**
The specific instruction for the specific asset being built right now. Generated by the Core Orchestrator's Brief Generator sub-agent from Documents 1–4.

*Format:* See Step 10 for the complete Asset Brief template. Every field must be populated before the brief is sent to the Builder agent.

## The Core Orchestrator — Activation Protocol

The Core Orchestrator Agent is the traffic controller of the entire Builder Layer. It receives the completed strategy documents, enforces the quality gates, generates asset briefs, and routes each brief to the correct Builder agent with the correct context pre-loaded.

### Activation Sequence

**Step 1 — Strategy Complete Check**
Core Orchestrator verifies all six quality gates from Step 10 are green:
- Gate 1: Expert Dossier `handoff_ready: true`
- Gate 2: Brand North Star Part 2 complete and quality-checked
- Gate 3: Growth Blueprint unit economics pass (LTV:CAC ≥ 3:1)
- Gate 4: Monthly Master Plan produced
- Gate 5: Monthly Messaging Brief produced (hook bank ≥ 10 concepts)
- Gate 6: Asset Brief template ready

If any gate is red, Core Orchestrator does not activate Builder agents. It surfaces the specific gate failure and routes back to the appropriate Layer 2 agent to resolve.

**Step 2 — Build Queue Assembly**
Core Orchestrator reads the Monthly Master Plan and assembles the full build queue — every asset needed for the month, in priority order, with assigned Builder agent for each.

**Priority order for build queue:**
1. Funnel assets (opt-in page, webinar/VSL, order page) — must exist before traffic is sent
2. Lead magnet content (PDF, quiz, or vibe-coded app spec)
3. Email sequences (welcome, pre-webinar, post-webinar — must be live before webinar runs)
4. Paid ad creative (first batch — enables paid traffic launch)
5. Organic content (first 2 weeks of the month)
6. Sales script (needed before first sales call)
7. Remaining organic content (weeks 3–4)
8. Additional paid creative variants

**Step 3 — Asset Brief Generation**
For each asset in the build queue, Core Orchestrator's Brief Generator sub-agent generates a fully populated Asset Brief by pulling from Documents 1–4.

No field in the Asset Brief is left blank. If a field cannot be populated from the strategy documents, Core Orchestrator flags it as a strategy gap and routes back to the Growth Strategy Agent before the brief is sent.

**Step 4 — Context Loading**
For each Builder agent activated, Core Orchestrator's Context Loader sub-agent pre-loads the relevant sections of the strategy documents into the Builder agent's context window.

Context loading is optimized — Builder agents don't receive the entire strategy stack for every asset. They receive:
- Full Document 2 (Brand North Star Part 2) — always
- Relevant sections of Document 1 (Voice DNA + Avatar section + Offer section always; additional sections based on asset type)
- Full Document 3 (Monthly Messaging Brief) — always
- Relevant section of Document 4 (the specific calendar entry for this asset) — always
- Full Document 5 (Asset Brief) — always

**Step 5 — Builder Agent Activation**
Core Orchestrator routes the brief and pre-loaded context to the correct Builder agent. The Builder agent confirms receipt of the complete input package before generating any output.

**Confirmation check (Builder agent runs before starting):**
```
PRE-GENERATION CHECKLIST
☐ Brand North Star Part 2 loaded — ALWAYS/NEVER rules reviewed
☐ Voice DNA sample extracts loaded — calibration standard set
☐ Monthly Messaging Brief loaded — hook bank and angle map available
☐ Asset Brief complete — all fields populated
☐ Compliance filter active for [expert's specialty]
☐ Proof anchors loaded — approved case studies available
Ready to generate: YES / NO
```

If any item is unchecked, the Builder agent returns an error to the Core Orchestrator rather than generating with incomplete context.

**Step 6 — Output Return and Review Routing**
Builder agent returns completed output to Core Orchestrator. Core Orchestrator routes to the Judge/Reviewer Agent (Layer 4) for quality and brand consistency check before the output reaches the expert.

## The Handoff Payload — Machine-Readable Format

In addition to the human-readable documents, the Core Orchestrator maintains a machine-readable handoff state object that tracks the status of every asset in the build queue.

```json
{
  "handoff_id": "[unique ID]",
  "expert_id": "[dossier ID]",
  "month": "[YYYY-MM]",
  "strategy_gates": {
    "gate_1_dossier_ready": true,
    "gate_2_brand_north_star": true,
    "gate_3_unit_economics": true,
    "gate_4_monthly_plan": true,
    "gate_5_messaging_brief": true,
    "gate_6_asset_brief_template": true,
    "all_gates_green": true
  },
  "build_queue": [
    {
      "asset_id": "[unique asset ID]",
      "asset_type": "[e.g. opt-in page]",
      "builder_agent": "[e.g. Funnel Builder Agent]",
      "priority": 1,
      "brief_status": "generated | pending | failed",
      "build_status": "queued | in-progress | complete | review | approved | deployed",
      "output_url": "[link to completed asset when built]",
      "review_status": "pending | passed | failed | revision-requested",
      "expert_approval": "pending | approved | revision-requested"
    }
  ],
  "documents_loaded": {
    "expert_dossier_brief": true,
    "brand_north_star_part2": true,
    "monthly_messaging_brief": true,
    "monthly_master_plan_brief": true
  },
  "compliance_filter_active": true,
  "specialty_compliance_rules": "[reference to compliance ruleset for this expert's specialty]"
}
```

This state object is updated in real time as assets move through the build → review → approval → deploy pipeline. The Analytics Dashboard (Layer 4) reads from this object to track production velocity and identify bottlenecks.

## Ongoing Handoff — Monthly Cycle

The handoff is not a one-time event. It repeats every month as the Monthly Master Plan and Monthly Messaging Brief are refreshed.

**Monthly handoff cycle:**

| Timing | Action |
|---|---|
| Month end (days 25–28) | Growth Strategy Agent produces next month's Monthly Master Plan and Monthly Messaging Brief |
| Month end (day 28) | Core Orchestrator reviews both documents, runs strategy gate checks, flags any gaps |
| Month end (day 29) | Core Orchestrator generates full build queue for next month |
| Month end (day 30) | Asset briefs generated for first-week priority assets |
| Month start (day 1) | Builder agents activated for first-week assets |
| Week 2 | Asset briefs generated for week 2–3 assets based on week 1 performance data |
| Mid-month | Performance review — if red/yellow flags emerge, Growth Strategy Agent issues course correction brief |
| Month end | Quarterly review triggers (every 3rd month) — full strategy refresh |
