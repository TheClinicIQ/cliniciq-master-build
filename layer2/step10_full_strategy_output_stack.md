## Purpose

This document defines the complete set of outputs the Layer 2 Strategy Agents produce — everything from the One Big Domino through the builder-level briefs that Layer 3 agents consume. It is the master reference for what "strategy complete" means in clinicIQ and what must exist before the Builder Layer is activated.

Step 10 is the bridge between strategy and execution. It defines every deliverable, its format, its owner, and exactly how it hands off to the Builder agent that consumes it.

---

## The Full Strategy Output Stack — Overview

Layer 2 produces two categories of output:

**Category 1 — Strategic Foundation Documents**
High-level, durable documents that guide the entire platform for this expert. Updated when the expert's positioning, offer, or avatar changes — not monthly.

**Category 2 — Operational Campaign Documents**
Specific, time-bound documents that guide the Builder agents for a specific period (monthly, quarterly, or campaign-specific). Updated on a recurring cadence.

Together these two categories give every Builder agent both the north star (what never changes) and the specific brief (what to build right now).

## Category 1 — Strategic Foundation Documents

### Output 1A — The Brand North Star Document
**Owner:** Brand Strategy Agent
**Format:** Two-part markdown document — Part 1 (narrative strategy) + Part 2 (operational rules)
**Cadence:** Built once at Layer 2 activation. Updated when positioning, mechanism, avatar, or offer stack changes materially.
**Consumed by:** All Layer 3 Builder agents (required read), Core Orchestrator Agent, Judge/Reviewer Agent (Layer 4)

**Must contain:**
- Brand positioning statement (Category of One format)
- One Big Domino statement (locked)
- Authority stack (top 3–4 signals ranked and deployment-mapped)
- Five-step differentiation argument
- Three-layer messaging architecture template + 5 written examples to Voice DNA standard
- Avatar messaging brief (condensed action brief for Builder agents)
- Brand voice consistency rules (approved frameworks, prohibited patterns, tone calibration table by content type)
- Mission and vision statements
- Mechanism demonstration direction
- Content pillar architecture (5–6 pillars with strategic purpose, false belief addressed, best content types)
- Part 2 Operational Rules (ALWAYS/NEVER lists, One Big Domino, positioning statement, avatar quick reference, mechanism, offer reference, proof anchors, content pillars, voice standard examples)

**Quality gate:** Cannot be marked complete until Part 2 Operational Rules section passes all quality standards defined in Step 8. Judge/Reviewer Agent runs automated quality check at completion.

### Output 1B — The Growth Marketing Blueprint
**Owner:** Growth Strategy Agent
**Format:** Structured strategy document — narrative + tables + calendar framework
**Cadence:** Built once at Layer 2 activation. 90-day plan refreshed quarterly. Long-term roadmap updated at major milestones.
**Consumed by:** Campaign Planner Agent (primary), Content Builder Agent, Funnel Builder Agent, Email & SMS Builder Agent, Ad Creative Builder Agent, Analytics Dashboard (Layer 4)

**Must contain:**
- Starting point profile (A/B/C/D) with rationale
- 90-day launch plan (week-by-week sequenced actions)
- Platform strategy (primary + secondary platforms, cadence, content mix per platform)
- Funnel architecture map (specific variant for this expert's awareness level)
- Funnel performance benchmarks (all 7 elements with red/yellow/green thresholds)
- Organic content growth plan (3 phases with cadence, mix, CTA rotation)
- Paid advertising framework (platform, objectives, creative structure, budget, scaling protocol)
- Email and SMS architecture (8 core sequences mapped with trigger, length, purpose)
- Lead magnet recommendation (type, topic, positioning, vibe-code spec if applicable, email follow-up sequence map)
- Webinar strategy (live vs. evergreen, frequency, optimization protocol)
- Referral and ascension system (timing, triggers, sequences)
- Competitive intelligence summary (market gaps, hook angles observed, differentiation opportunities)
- Unit economics stress test (LTV, CAC, LTV:CAC ratio, break-even, pessimistic/optimistic scenarios)
- KPI definitions (metrics tracked, benchmarks, red/yellow/green thresholds for Layer 4 dashboard)
- 30/60/90-day milestones

**Quality gate:** Cannot be marked complete until the unit economics stress test passes (LTV:CAC ≥ 3:1 in pessimistic scenario). If it fails, the Growth Strategy Agent flags the specific variable causing the failure and proposes the fix before the blueprint is finalized.

## Category 2 — Operational Campaign Documents

### Output 2A — The Monthly Master Plan
**Owner:** Growth Strategy Agent (Monthly Planner Sub-Agent)
**Format:** Two-part document — monthly calendar view + master brief
**Cadence:** Produced at the start of every month. Updated mid-month if performance data triggers a significant course correction.
**Consumed by:** All Layer 3 Builder agents (required read at start of each month), Core Orchestrator Agent

**Must contain:**

**Part 1 — Monthly Calendar:**
- Every social post for the month: platform, date, content pillar, format, hook angle, CTA type
- Video production schedule: which scripts are needed and when
- Lead magnet status: which is primary CTA this month, any new launch
- Webinar schedule: live dates, replay windows, promotional push dates
- Paid campaign schedule: campaigns active, creative refresh dates, A/B test timelines
- Email broadcast schedule: send dates, subject line concepts, primary CTA
- SMS broadcast schedule
- Seasonal flag: any peak buying moment this month that requires a push

### Output 2B — The Monthly Messaging Brief
**Owner:** Growth Strategy Agent (Messaging Brief Writer Sub-Agent) — inputs from Brand North Star Document
**Format:** Structured reference document
**Cadence:** Produced monthly alongside the Monthly Master Plan. Updated when new organic performance data warrants hook or angle rotation.
**Consumed by:** All Layer 3 Builder agents as the campaign-level creative brief

**Must contain:**

**Hook Bank (updated monthly):**
20–30 hook concepts organized by category. Each hook includes:
- The hook text (written to Voice DNA standard)
- Category (pain / villain / mechanism / identity / curiosity / proof)
- Source (organic-tested + performance data if available / agent-generated / expert-submitted)
- Status (active-test / proven-winner / retired)

**Angle Map:**
For each active content pillar this month — 5–7 specific angles. Not topics. Specific entry points.

Example format:
```
PILLAR: Root Cause Education
Angle 1: The specific moment at 2pm when the crash hits — what's happening biologically that conventional medicine never addresses
Angle 2: Why "normal" lab ranges were set using sick populations — and what optimal actually looks like
Angle 3: The three upstream immune triggers that thyroid medication doesn't touch
Angle 4: What a 90-day elimination of the root cause actually feels like day by day
Angle 5: The research paper your endocrinologist has never read (and what it proves)
```

**Objection Response Library:**
Every primary objection with:
- The objection (exact words the avatar uses)
- Emotional beat to hit first (acknowledge before arguing)
- Logical argument (the evidence or mechanism explanation)
- Proof element to close (the specific case study or testimonial that seals it)
- Recommended deployment (email sequence / webinar close / sales script / FAQ)

**Emotional Beat Sequence:**
The specific emotional journey for this month's primary campaign:
1. Current pain state (where the avatar is right now)
2. Validated and understood (the moment they feel seen)
3. The villain named (what's actually been causing this)
4. Possibility opened (the crack in the belief that nothing will work)
5. Mechanism revealed (the new vehicle introduced with demonstration)
6. Transformation visualized (future pacing into the transformation identity)
7. Decision primed (urgency + offer + guarantee + CTA)

Each piece of content in the month is mapped to one point on this journey. No piece is disconnected from the sequence.

**Social Proof Deployment Map:**
Which specific case studies and testimonials get used where this month:
- Primary webinar proof story: [Case study name/alias]
- Primary ad testimonial: [Case study name/alias + format]
- Primary email proof moment: [Case study name/alias + which email in which sequence]
- Primary organic proof content: [Case study name/alias + which post in the calendar]

**Campaign-Specific Language:**
New vocabulary or framings being tested this month. Tracked for performance and fed back to Voice DNA enrichment if proven.

### Output 2C — Builder-Level Asset Briefs
**Owner:** Core Orchestrator Agent (routes from Monthly Master Plan + Monthly Messaging Brief)
**Format:** Individual brief per asset, generated on demand or in batch at month start
**Cadence:** Generated when a Builder agent is activated for a specific asset
**Consumed by:** The specific Builder agent assigned to the asset

Every asset produced in clinicIQ is preceded by a brief. The brief is the Core Orchestrator's translation of the strategy documents into a specific, actionable instruction for a specific Builder agent.

**Standard Asset Brief format:**

```

Builder agent: [Content Builder / Ad Creative Builder / Email Builder / Funnel Builder / etc.]
Expert: [Name]
Month: [Month/Year]

CONTEXT (read before generating):
- Brand North Star Part 2: [Attached / Link]
- Monthly Messaging Brief: [Attached / Link]
- Voice DNA sample extracts: [Attached]

THIS ASSET:
Purpose: [What this asset does in the funnel]
Platform/channel: [Where it appears]
Content pillar: [Which pillar it belongs to]
Hook angle: [Specific hook from the hook bank]
CTA type: [Specific CTA from the CTA rotation]
Webinar language variant: [training / masterclass / workshop / etc.]
Proof element to include: [Specific case study or testimonial if applicable]
Emotional beat target: [Which point in the emotional beat sequence this asset hits]
```

## The Strategy-to-Builder Handoff Flow

This is the complete sequence from strategy completion to Builder agent activation:

```
LAYER 1 COMPLETE
Expert Dossier (164 fields) + Silent Diagnosis payload
        ↓
LAYER 2 ACTIVATION
Brand Strategy Agent + Growth Strategy Agent run in parallel
        ↓
OUTPUT 1A: Brand North Star Document (Brand Strategy Agent)
OUTPUT 1B: Growth Marketing Blueprint (Growth Strategy Agent)
        ↓
Core Orchestrator Agent loads both documents
        ↓
OUTPUT 2A: Monthly Master Plan (Growth Strategy Agent — Monthly Planner Sub-Agent)
OUTPUT 2B: Monthly Messaging Brief (Growth Strategy Agent — Messaging Brief Writer Sub-Agent, reads Brand North Star)
        ↓
OUTPUT 2C: Asset Briefs (Core Orchestrator routes from 2A + 2B)
        ↓
LAYER 3 BUILDER AGENTS ACTIVATE
Each Builder agent receives: Expert Dossier Brief + Brand North Star Part 2 + Monthly Messaging Brief + Asset Brief
        ↓
ASSETS PRODUCED
        ↓
Judge/Reviewer Agent (Layer 4) checks against Brand North Star
        ↓
Expert reviews and approves
        ↓
Analytics Dashboard tracks performance
        ↓
OUTPUT 2D: Quarterly Strategy Review (feeds back to update Outputs 1A, 1B, 2A, 2B)
        ↓
FLYWHEEL CONTINUES
```

## Strategy Output Quality Gates — Summary

Before any Builder agent is activated, the following must be true:

| Gate | Required Output | Owner | Status Check |
|---|---|---|---|
| Gate 1 | Expert Dossier `handoff_ready: true` | Onboarding Agent | Auto-checked at Layer 2 activation |
| Gate 2 | Brand North Star Part 2 complete | Brand Strategy Agent | Judge/Reviewer quality check |
| Gate 3 | Growth Blueprint unit economics pass | Growth Strategy Agent | LTV:CAC ≥ 3:1 in pessimistic scenario |
| Gate 4 | Monthly Master Plan produced | Monthly Planner Sub-Agent | All calendar fields populated |
| Gate 5 | Monthly Messaging Brief produced | Messaging Brief Writer Sub-Agent | Hook bank ≥ 10 concepts, angle map complete |
| Gate 6 | Asset Brief generated | Core Orchestrator | All brief fields populated |

No Builder agent generates output without Gates 1–6 passing. This is enforced by the Core Orchestrator Agent — it will not route an asset request to a Builder agent until all gates are green.

## What Layer 3 Builder Agents Receive — The Complete Input Package

Every Builder agent, for every asset, receives the following package before generating any output:

1. **Expert Dossier Brief** — who the expert is, their voice, their avatar, their offer
2. **Brand North Star Document — Part 2** — the operational rules (ALWAYS/NEVER, One Big Domino, positioning, avatar quick reference, mechanism, offer, proof anchors, content pillars, voice examples)
3. **Monthly Messaging Brief** — hook bank, angle map, objection library, emotional beat sequence, proof deployment map
4. **Monthly Master Plan — Brief** — what's being built this month, primary CTA, webinar details, active campaigns
5. **Asset Brief** — the specific instruction for this specific asset

With this package, every Builder agent has everything it needs to produce on-brand, on-strategy, on-voice output without any additional prompting. The quality of the input package determines the quality of the output. The strategy layer's job is to make the Builder layer's job effortless.
