## What a Workflow IS

A **Workflow** is a named, multi-step, coordinated growth operation that chains multiple builders together toward a single business outcome. It is NOT a single builder run. It is NOT a Zapier-style automation trigger. It is a complete growth engine — a sequenced set of builder tasks where the output of each step feeds the next, all coordinated by IQ Claw.

**The distinction:**
- **Builder** = "Make me a thing" (one builder, one asset type)
- **Workflow** = "Build me an engine" (multiple builders, coordinated, producing a complete business system)

A Workflow is the difference between "write me 7 social posts" and "build me an evergreen webinar funnel with opt-in pages, a webinar script, pre-webinar email sequences, post-webinar follow-up, ad creatives driving cold traffic, and a consult close script — all connected, all in my voice, all targeting the right avatar."

**Why this matters:** The 12 Builders are the parts. Workflows are the machine. Most health experts don't know they need a machine — they think they need a post. Workflows show them what a real growth engine looks like and let them build one with a single conversation.

## How Workflows Work — The Expert's Experience

### Step 1: Initiation
Three entry points:

**A. "+ New Workflow" button** → Opens IQ Claw chat (center panel) pre-seeded:
> "What do you want to build? Describe the business outcome you're after — not the pieces, the result. For example: 'I want a funnel that turns cold traffic into booked consults' or 'I want a full content machine that runs every month without me.' I'll design the engine and show you the blueprint before we build anything."

**B. Template selection** → Expert clicks a pre-built engine template → IQ Claw opens with the template blueprint already drafted, personalized to the expert's dossier:
> "Here's your Evergreen Webinar Funnel blueprint — customized to your Voice DNA, your avatar Stressed Mom Sarah, and your Signal Reset Protocol mechanism. Here's what we're building: [blueprint]. Want to adjust anything before I start?"

**C. IQ Claw conversation** → Expert describes what they want in normal chat ("I need to get more booked calls") → IQ recognizes this as a Workflow-level request and proposes a blueprint:
> "That sounds like you need a full conversion engine. Here's what I'd build: [blueprint]. This is a Workflow — multiple builders working together. Want me to save this as a Workflow and start building?"

### Step 2: The Blueprint
Before any builder fires, IQ Claw presents the **Workflow Blueprint** — a visual plan the expert reviews and approves. The blueprint shows:

- **Workflow name** (auto-generated, expert can rename)
- **Business objective** — one sentence describing the outcome
- **Steps in order** — numbered, each with:
  - Which builder handles it
  - What it produces
  - What inputs it needs (from dossier, from previous steps, or from the expert)
  - Estimated time
- **Dependencies** — which steps must complete before others can start (e.g., webinar script must finish before post-webinar emails can reference specific talking points)
- **Parallel tracks** — which steps can run simultaneously (e.g., ad creatives and funnel pages can build at the same time)
- **Total estimated time** — sum of sequential steps (parallel steps don't add to total)
- **Assets produced** — complete list of deliverables when the workflow finishes

**Blueprint display:** A clean vertical step list inside the right panel (or a modal on mobile). Each step is a compact card showing builder icon, step name, and status. Dependencies shown as connecting lines. Parallel steps shown side-by-side.

### Step 3: Expert Review
The expert can:
- **Approve as-is** → "Build it"
- **Add steps** → "Also add a podcast episode promoting the webinar"
- **Remove steps** → "I don't need the challenge sequence"
- **Reorder** → "Build the email sequences before the funnel pages"
- **Adjust scope** → "Make it 14 emails instead of 7" or "Target Weekend Warrior Mike instead of Stressed Mom Sarah"
- **Change builder inputs** → "Use my premium offer instead of core" or "Use the new mechanism name"

IQ Claw updates the blueprint in real-time as the expert makes changes.

### Step 4: Execution
Expert clicks **"Start Building"** → Builders fire in sequence, respecting dependencies. Parallel tracks run simultaneously.

**The right panel transforms into the Workflow Progress View:**
- Workflow name + overall progress bar at top
- Each step shown as a card with:
  - Builder icon + step name
  - Status: ⏳ Queued → 🔄 Building → ✅ Complete
  - Progress bar (for steps with multiple assets)
  - [View Output →] link when complete
- Currently active step(s) highlighted with a subtle pulse animation
- IQ Claw narrates progress in the chat panel:
  > "Funnel pages are done — 5 pages built. Webinar script is generating now. I'll start the email sequences once the script locks so I can reference your specific talking points."

### Step 5: Review & Edit
As each step completes, the expert can review the output immediately:
- Click [View Output →] on any completed step → right panel shows the assets
- Edit via IQ Claw chat: "Change the hook on the opt-in page" → IQ routes the edit to the correct builder → right panel updates
- The workflow continues building subsequent steps while the expert reviews earlier ones — no need to wait

### Step 6: Completion
When all steps finish:
- Overall progress bar hits 100%
- IQ Claw delivers a completion summary in chat:
  > "Your Evergreen Webinar Funnel is complete. Here's what's ready: 5 funnel pages, 1 webinar script with 47 slides, 18 emails across 4 sequences, 9 ad creatives in 3 hook angles, and 1 consult close script. Everything is saved to IQ Drive under /Workflows/Evergreen Webinar Funnel/. Want to review anything before we go live?"
- Right panel shows the complete asset inventory with links to each deliverable
- **[Save to Drive]** — all assets organized in a workflow-specific folder
- **[Activate]** — for assets that can go live (schedule content, activate email sequences, launch ads)

### Step 7: Persistence
The workflow persists in "My Workflows" as a named, living engine. It's not a one-time run. The expert can:

- **Rerun** — regenerate all assets with current strategy, voice, and performance data. "Refresh everything for May."
- **Rerun one step** — regenerate just the ad creatives (because of creative fatigue) without touching the rest
- **Optimize** — Layer 4 Intelligence flags underperformers: "Your webinar show rate is 18%. I recommend refreshing the pre-webinar email sequence. Want me to rebuild just those 5 emails?" The expert approves → only that step reruns
- **Clone** — duplicate the entire workflow for a different offer, avatar, or campaign. "Clone this for Weekend Warrior Mike instead of Stressed Mom Sarah."
- **Schedule** — set recurring reruns. "Refresh ad creatives every 3 weeks." "Generate a new content batch every month."
- **View performance** — Layer 4 data tied to the workflow's assets. See which funnel pages convert, which emails get opened, which ad hooks win — all in context of the workflow that produced them.
- **Archive** — when a workflow is no longer active, archive it (still viewable, just not in the active list)

## Custom Workflows — When the Expert Builds Their Own

Beyond pre-built templates, the expert can build custom workflows for any multi-step operation they can describe. IQ Claw is the architect.

**How it works:**

The expert describes the outcome: "I want to build a joint venture webinar with another practitioner" or "I need a complete rebrand — new avatar, new messaging, new content" or "Build me a Black Friday campaign."

IQ Claw:
1. Identifies which builders are needed
2. Determines the dependency order
3. Identifies what inputs it needs from the expert (vs. what it can pull from the dossier)
4. Presents the blueprint
5. Asks for approval
6. Executes

**Examples of custom workflows:**

- **"Rebrand for new avatar"** → Voice DNA refresh + new avatar extraction + Brand North Star update + new messaging brief + new content batch + new ad creatives
- **"Launch a second offer tier"** → Offer architecture update + new funnel page + new webinar section + new email sequence + new ad creative angle
- **"Holiday flash sale"** → Flash offer design + 5 emails (3-day urgency sequence) + 5 social posts (countdown) + 3 ad creatives
- **"Respond to a trending topic"** → Trend analysis + 7 social posts riding the trend with expert's mechanism angle + 1 email broadcast
- **"Rebuild my worst-performing funnel step"** → Layer 4 identifies the weak link + specific builder reruns with optimized messaging

## Workflow States

Every workflow has a lifecycle:

| State | What It Means | Visual |
|-------|---------------|--------|
| **Draft** | Blueprint created, not yet started | Gray, dashed border |
| **Running** | Actively building — one or more steps in progress | Green pulse, progress bar animating |
| **Paused** | Expert paused execution (can resume) | Amber, pause icon |
| **Needs Review** | All steps complete, awaiting expert review of outputs | Blue, notification badge |
| **Complete** | Reviewed and approved by expert | Green checkmark |
| **Scheduled** | Set to auto-run on a cadence | Clock icon + next run date |
| **Optimizing** | Layer 4 flagged an underperformer — optimization recommended | Amber sparkle icon + specific recommendation |
| **Archived** | No longer active, kept for reference | Gray, moved to archive |

## Relationship to Other UI Surfaces

**Builders tab (right panel in IQ workspace):**
Still exists. Still shows the 12 builder cards for single-asset quick launches. This is for "make me a post" or "write me an email." Quick, single-builder work.

**Workflows page (sidebar):**
For multi-step engines. The big-picture, coordinated operations. Different intent, different scope.

**IQ Claw chat:**
The bridge between both. When an expert describes something in chat, IQ decides whether it's a single builder task or a workflow-level operation and routes accordingly. If it's ambiguous: "That sounds like it might need a full engine — want me to build a Workflow for this, or just run the Content Builder for a quick batch?"

## Naming: "Growth Engines" vs. "Workflows"

**Recommendation:** Call them **"Growth Engines"** on all user-facing surfaces. "Workflows" is a SaaS word. "Growth Engines" is a business-outcome word. It signals: "this builds your business" not "this automates a task."

- Sidebar label: "Engines" (with the workflow/gear icon)
- Page heading: "Growth Engines"
- Button: "+ New Engine"
- Tab: "Engine Library"
- IQ Claw language: "I'll build you an engine for that" / "Your webinar engine is running" / "Let me spin up a campaign engine"

The internal/technical term stays "workflow" in code and API. The user-facing language is always "engine."
